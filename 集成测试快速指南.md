# DrawAnywhere 集成测试快速指南

## 🚀 快速开始

### 1. 前置要求

- ✅ Android Studio 已安装
- ✅ Android SDK API 26+ 已安装
- ✅ 真实设备已连接 **或** 模拟器已启动

### 2. 同步 Gradle

```bash
./gradlew clean build
```

### 3. 运行所有集成测试

```bash
./gradlew connectedAndroidTest
```

## 🐛 常见问题

### Q1: 测试失败，提示 "No connected devices"

**解决方案**:
1. 连接真实设备并启用 USB 调试
2. 或启动 Android 模拟器

### Q2: 测试因权限问题失败

**解决方案**:
在设备上手动授予悬浮窗权限：
```
设置 → 应用 → DrawAnywhere → 高级 → 在其他应用上层显示 → 允许
```

### Q3: IDE 显示 Unresolved reference 错误

**解决方案**:
1. 点击 `File → Sync Project with Gradle Files`
2. 或运行 `./gradlew --refresh-dependencies`

### Q4: 协程测试超时

**解决方案**:
检查是否正确使用了 `testDispatcher.scheduler.advanceUntilIdle()`

## 💡 编写新测试的建议

### 模板

```kotlin
@OptIn(ExperimentalCoroutinesApi::class)
class MyIntegrationTest {

    private lateinit var viewModel: DrawViewModel
    private lateinit var controller: DrawController
    private lateinit var mockPreferencesMgr: PreferencesManager
    private val testDispatcher = StandardTestDispatcher()

    @Before
    fun setup() {
        Dispatchers.setMain(testDispatcher)
        
        controller = DrawController()
        controller.setPenConfig(PenConfig(penType = PenType.Pen, color = Color.Red, width = 5f))
        
        mockPreferencesMgr = mockk(relaxed = true)
        every { runBlocking { mockPreferencesMgr.getSavedUiState() } } returns UiState()
        every { runBlocking { mockPreferencesMgr.getSavedServiceState() } } returns ServiceState()

        viewModel = DrawViewModel(
            controller = controller,
            preferencesMgr = mockPreferencesMgr,
            initialUiState = UiState(),
            initialServiceState = ServiceState(),
            stopService = { }
        )
    }

    @After
    fun tearDown() {
        Dispatchers.resetMain()
    }

    @Test
    fun testMyFeature() = runBlocking {
        // Given
        
        // When
        
        // Then
    }
}
```

### 最佳实践

1. ✅ 使用真实的 `DrawController`
2. ✅ Mock `PreferencesManager` 避免磁盘 I/O
3. ✅ 使用 `testDispatcher.scheduler.advanceUntilIdle()` 推进协程
4. ✅ 直接访问 StateFlow 的 `.value` 进行断言
5. ✅ 每个测试方法专注于一个场景
