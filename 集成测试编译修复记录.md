## 问题 2: META-INF/LICENSE.md 重复文件冲突

### 错误信息
```
> Task :app:mergeDebugAndroidTestJavaResource FAILED

FAILURE: Build failed with an exception.

* What went wrong:
Execution failed for task ':app:mergeDebugAndroidTestJavaResource'.
> A failure occurred while executing com.android.build.gradle.internal.tasks.MergeJavaResWorkAction
   > 6 files found with path 'META-INF/LICENSE.md' from inputs:
      - org.junit.jupiter:junit-jupiter-params:5.8.2/junit-jupiter-params-5.8.2.jar
      - org.junit.jupiter:junit-jupiter-engine:5.8.2/junit-jupiter-engine-5.8.2.jar
      - org.junit.jupiter:junit-jupiter-api:5.8.2/junit-jupiter-api-5.8.2.jar
      - org.junit.platform:junit-platform-engine:1.8.2/junit-platform-engine-1.8.2.jar
      - org.junit.platform:junit-platform-commons:1.8.2/junit-platform-commons-1.8.2.jar
      - org.junit.jupiter:junit-jupiter:5.8.2/junit-jupiter-5.8.2.jar
```

### 根本原因
多个 JUnit 5 依赖包中都包含相同的 `META-INF/LICENSE.md` 文件，Android Gradle Plugin 在合并资源时检测到冲突。

涉及的依赖：
- junit-jupiter-params:5.8.2
- junit-jupiter-engine:5.8.2
- junit-jupiter-api:5.8.2
- junit-platform-engine:1.8.2
- junit-platform-commons:1.8.2
- junit-jupiter:5.8.2

### 解决方案
在 `app/build.gradle.kts` 中添加 packaging 配置，排除冲突的元数据文件：

```kotlin
android {
    // ... 其他配置
    
    packaging {
        resources {
            excludes += "/META-INF/{AL2.0,LGPL2.1,LICENSE*.md}"
        }
    }
}
```

**说明**:
- `AL2.0`: Apache License 2.0
- `LGPL2.1`: GNU Lesser General Public License 2.1
- `LICENSE*.md`: 所有 LICENSE.md 文件

这是 Android 项目中处理依赖元数据冲突的标准做法，不会影响应用的正常运行。

## IDE 错误提示说明

即使在编译成功后，IDE（Android Studio）可能仍然显示一些 "Unresolved reference" 错误，这是正常的。原因：

1. **IDE 索引延迟**: IDE 需要时间同步 Gradle 依赖
2. **缓存问题**: IDE 缓存可能未及时更新

### 解决方法

如果 IDE 仍显示错误，可以尝试：

1. **同步 Gradle**:
   ```
   File → Sync Project with Gradle Files
   ```

2. **清理缓存**:
   ```
   File → Invalidate Caches / Restart
   ```

3. **重新导入项目**:
   ```
   File → Close Project → Reopen
   ```

**重要**: 只要命令行编译成功（`BUILD SUCCESSFUL`），就可以忽略 IDE 的错误提示，直接运行测试。

## 经验总结

### 最佳实践

1. **始终检查导入语句**: 创建新测试文件时，确保所有使用的类都已正确导入
2. **处理依赖冲突**: 当添加新的测试依赖时，可能需要配置 packaging excludes
3. **先命令行验证**: 使用命令行编译验证，不要完全依赖 IDE 的错误提示

### 常见陷阱

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| 缺少导入 | Unresolved reference 'XXX' | 添加对应的 import 语句 |
| 依赖冲突 | Duplicate files during packaging | 配置 packaging.resources.excludes |
| IDE 误报 | 编译成功但 IDE 显示错误 | 同步 Gradle 或清理缓存 |
